Goals¶

The purpose of the www.chanceme.info website is allow students and parents a simple interface with which to predict the probability of getting into a given school. The site allows 'what-if' scenarios so a student can determine the effect on the probability by changing her profile and actions.

We feel that this is an important service lacking to the four million college applicants every year. Students try expensive and time-consuming extra-curricular activities, with the purpose of bolstering their college application, without any knowledge of the impact it will have. Our data shows that GPA and standardized tests have a far larger impact than extra-curricalar activities. The purpose of the website is allowing students to see this, as well as other insights, for themselves.

Additionally, it allows the student to prioritize their college applications. We see that applying early is more important to some schools than to others, so the student is better off rushing to apply early to the school where it will make a more significant difference.

Features¶

  • Reactive site, requiring no form submissions
  • Mobile friendly, works with popular mobile devices (although this has not been extensively tested)
  • Plots of candidate placement vs historical competition
  • Criteria comparison: allows the student to see relative importance of one factor against another

Implementation¶

As seen in the diagram at the top of this page, the website consists of three major subsystems:

  • a static HTML site (hosted at SquareSpace)
  • an interactive iframe which allows users to enter their information and see predictions and analysis (written in R / Shiny, hosted at http://www.shinyapps.io/)
  • a webservice that produces predicted probabilities for a given candidate (written in Python / Flask, hosted on OpenShift)

This implementation allowed us to parallelize development time as different people could work on different subsystems while minimizing development type by re-using code whenever possible. The prediction webservice takes the knowledge gleaned from our classification research and puts it into production using the same Scikit-learn libraries and hyperparameters.

For extensive details about the implementation and deployment, see Appendix B - Build and Deployment.
